# 需求讨论记录

## 项目目标

创建一个面向新手的 Agent 使用教学前端网站，特点：
- 从入门安装到进阶实战的完整教学路径
- 交互式引导，降低新手学习门槛
- 清新友好的设计风格

## 技术决策

### 1. 技术栈选择：Vue 3 + Vite

**讨论过程：**
- 用户不熟悉现代前端框架，需要沟通选择
- 解释了 React/Vue 都是纯前端方案，不需要后端服务
- Vite 只是打包工具，产物是静态 HTML/CSS/JS 文件

**选择理由：**
- Vue 模板语法更接近原生 HTML，学习成本较低
- 响应式系统适合处理交互式教学步骤
- 生态成熟，便于后续扩展

### 2. 交互形式：自由进度制

**讨论过程：**
- 用户希望步骤勾选后进入下一步，保持用户注意力
- 提供了三种选项：严格步骤制、自由进度制、混合模式

**最终选择：自由进度制**
- 用户可以跳步，不强制顺序
- 完成时有鼓励反馈（如 "太棒了！你已经完成了..."）
- 顶部进度条显示完成百分比

### 3. 进阶内容：项目实战案例

**内容定位：**
- 展示用户自己的 Agent 项目是怎么做的
- 包含关键代码片段和运行结果
- 代码块支持语法高亮

### 4. 设计风格：清新友好

**设计方向：**
- 浅色为主，柔和的蓝绿色调
- 减少技术感压迫，降低门槛感
- 宽松舒适的间距，信息密度适中

## 内容结构

### 入门教程
1. 安装 Node.js
2. 安装 Claude Code CLI
3. 配置 API Key 环境变量
4. 打开终端/控制台
5. 运行第一个命令
6. 理解基本输出

### 进阶实战
- 用户自己的 Agent 项目案例
- 关键代码和实现思路
- 运行结果展示

### 小技巧分享
- 卡片式布局
- 分类标签（效率、调试、配置等）
- 简洁实用

## 核心组件

- **StepCard.vue**: 步骤卡片，包含勾选框和鼓励反馈
- **StepProgress.vue**: 进度指示器
- **ProjectCard.vue**: 项目展示卡片
- **CodeBlock.vue**: 代码高亮块

## 部署方案

暂定静态部署：
- GitHub Pages（免费，自动部署）
- 或 Vercel（零配置，速度快）
